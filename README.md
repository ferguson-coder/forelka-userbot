## 📖 О проекта

**Forelka Userbot** — это многофункциональный Telegram userbot, разработанный для автоматизации задач и расширения возможностей Telegram. Проект предоставляет гибкую модульную архитектуру, позволяющую легко добавлять новые функции и настраивать бота под свои нужды.

### Ключевые возможности

| Возможность | Описание |
|-------------|----------|
| 🧩 **Модульность** | Загрузка модулей из репозиториев и файлов |
| 🤖 **Инлайн-бот** | Встроенный Telegram bot для управления |
| 📊 **Management Group** | Группа с топиками для логов и команд |
| 🔐 **Система овнеров** | Управление правами доступа |
| ⚙️ **Гибкая конфигурация** | Per-account конфиги с алиасами |
| 📦 **Автоустановка зависимостей** | Pip-пакеты для модулей |
| 🌐 **Веб-авторизация** | HTML-страница для входа в Telegram |
| 🔗 **Туннелирование** | Публичный доступ через localhost.run |

## 🔑 Получение API ключей Telegram

Для работы userbot необходимы API ключи Telegram.

### Пошаговая инструкция

1. **Перейдите на портал разработчика**
   
   Откройте [my.telegram.org](https://my.telegram.org) в браузере.

2. **Авторизуйтесь**
   
   Введите номер телефона в международном формате (например, `+79991234567`).
   
   Получите код подтверждения в Telegram и введите его на сайте.

3. **Создайте приложение**
   
   - Перейдите в раздел **API development tools**
   - Нажмите **Create application**
   - Заполните форму:
     - **App title**: Название приложения (любое)
     - **Short name**: Короткое имя (любое)
     - **URL**: Можно оставить пустым
     - **Platform**: Выберите любую

4. **Получите ключи**
   
   После создания вы получите:
   - **App api_id**: Числовой идентификатор
   - **App api_hash**: Строка хэша

> **Warning:** Никогда не передавайте API ключи третьим лицам! При компрометации отзовите их на my.telegram.org и создайте новые.

## 🚀 Быстрый старт

### Запуск бота

```bash
python main.py
```

### Первый запуск

При первом запуске вам будет предложено выбрать способ авторизации:

```
No session found.
Choose login method:
  1) Terminal (API ID/HASH + phone in terminal)
  2) Web panel (HTML login page)
  3) Web + tunnel (HTML + public localhost.run URL)
>
```

#### Способ 1: Terminal (рекомендуется)

```bash
> 1
API ID: 12345678
API HASH: abcdef0123456789
```

После ввода ключей:
1. Введите номер телефона
2. Получите код подтверждения в Telegram
3. Введите код (если включена 2FA — введите пароль)

#### Способ 2: Web panel

```bash
> 2
Web panel: http://127.0.0.1:8000
```

1. Откройте `http://127.0.0.1:8000` в браузере
2. Пройдите авторизацию через веб-интерфейс
3. Сессия сохранится автоматически

#### Способ 3: Web + Tunnel

```bash
> 3
Web panel: http://127.0.0.1:8000
Public URL: https://abc123.localhost.run
```

1. Откройте публичный URL с любого устройства
2. Пройдите авторизацию
3. Удобно для входа с телефона/планшета

### После авторизации

Бот автоматически:
1. ✅ Создаст сессию `forelka-<user_id>.session`
2. ✅ Сохранит API ключи в защищённом файле
3. ✅ Создаст Management Group с топиками
4. ✅ Настроит инлайн-бота

### Проверка работы

Отправьте в Telegram (в «Избранное» или любой чат):

```
.ping
```

Если бот ответил `Pong!` с задержкой — всё работает! 🎉

## 🧩 Модульная система

### Архитектура модулей

Forelka использует гибкую систему загрузки модулей с поддержкой:

- **Системные модули** (`modules/`) — базовые функции
- **Пользовательские модули** (`loaded_modules/`) — дополнительные функции
- **Модули из репозиториев** — загрузка из GitHub

### Структура модуля

```python
# Метаданные (опционально)
__meta__ = {
    "name": "MyModule",
    "version": "1.0.0",
    "author": "Author Name",
    "description": "Описание модуля",
    "commands": ["cmd1", "cmd2"],
    "requires": ["aiohttp", "pillow>=1.0"]
}

# Команда
async def cmd1_cmd(client, message, args):
    """Описание команды"""
    await message.edit("Hello!")

# Регистрация
def register(app, commands, module_name, kernel=None):
    commands["cmd1"] = {"func": cmd1_cmd, "module": module_name}
```

### Формат метаданных

| Поле | Тип | Описание |
|------|-----|----------|
| `name` | string | Название модуля |
| `version` | string | Версия (semver) |
| `author` | string | Автор модуля |
| `description` | string | Краткое описание |
| `commands` | list | Список команд |
| `requires` | list | Pip-зависимости |

### Зависимости модулей

Модуль может указать зависимости несколькими способами:

```python
# Вариант 1: В __meta__
__meta__ = {
    "requires": ["aiohttp", "pillow>=1.0"]
}

# Вариант 2: Отдельная переменная
__requires__ = ["aiohttp", "pillow"]

# Вариант 3: Строка
__requires__ = "aiohttp, pillow"
```

Зависимости устанавливаются **автоматически** при загрузке модуля.

### Сигнатуры register функции

Модуль может иметь разные сигнатуры `register`:

```python
# Полная (4 параметра)
def register(app, commands, module_name, kernel):
    ...

# Стандартная (3 параметра)
def register(app, commands, module_name):
    ...

# Минимальная (2 параметра)
def register(app, commands):
    ...

# Без параметров
def register():
    ...
```

### Загрузка модулей

#### Из репозитория

```bash
# Добавить репозиторий
.addrepo https://github.com/user/repo

# Скачать модуль
.dlm weather

# Скачать по прямой ссылке
.dlm https://raw.githubusercontent.com/user/repo/main/weather.py
```

#### Из файла

1. Отправьте `.py` файл в чат с ботом
2. Ответьте командой `.lm`

#### Удаление модуля

```bash

```

> **Warning:** Системные модули из папки `modules/` защищены от удаления.

## ⚙️ Конфигурация

### Файлы конфигурации

Проект использует per-account конфигурацию — каждый аккаунт имеет свои файлы.

| Файл | Назначение |
|------|------------|
| `config-<user_id>.json` | Конфигурация аккаунта |
| `kernel_config-<user_id>.json` | Настройки ядра и бота |
| `repos.json` | Список репозиториев модулей |
| `forelka_config.db` | SQLite база конфигураций |

### Конфигурация аккаунта

Основные параметры:

| Параметр | Тип | Описание |
|----------|-----|----------|
| `management_group_id` | integer | ID группы управления |
| `management_topics` | object | ID топиков (Логи, Команды, etc.) |
| `owners` | array | Список ID владельцев |
| `aliases` | object | Алиасы команд |
| `prefix` | string | Префикс команд |

### Конфигурация ядра

| Параметр | Тип | Описание |
|----------|-----|----------|
| `inline_bot_token` | string | Токен инлайн-бота |
| `inline_bot_username` | string | Username инлайн-бота |
| `api_id` | integer | API ID Telegram |
| `api_hash` | string | API HASH Telegram |

### Управление конфигурацией

#### Смена префикса

```bash
.prefix !
# Теперь команды: !help, !ping, и т.д.
```

#### Создание алиаса

```bash
.alias p ping
# Теперь .p работает как .ping
```

#### Управление овнерами

```bash
# Добавить овнера
.addowner 123456789

# Удалить овнера
.delowner 123456789

# Список овнеров
.owners
```

> **Note:** Конфигурационные файлы содержат чувствительные данные. Не передавайте их и не коммитьте в Git.

## 🔧 Troubleshooting

### Ошибки при запуске

#### `ModuleNotFoundError: No module named 'telethon'`

```bash
pip install telethon
```

#### `Session file not found`

Удалите повреждённые сессии и запустите заново:

```bash
rm forelka-*.session
python main.py
```

#### `API ID/HASH не найдены`

Проверьте наличие файлов конфигурации или введите ключи заново при запуске.

### Ошибки модулей

#### Модуль не загружается

Проверьте:
1. Наличие функции `register`
2. Правильную сигнатуру `register`
3. Отсутствие синтаксических ошибок

```bash
python -m py_compile loaded_modules/module.py
```

#### Зависимости не устанавливаются

```bash
# Ручная установка
.pip install <package>

# Или через систему
pip install <package>
```

### Ошибки инлайн-бота

#### Бот не отвечает

1. Проверьте токен в конфигурации
2. Убедитесь, что бот добавлен в овнеры
3. Перезапустите бота: `.restart`

#### Ошибка "Доступ запрещен"

Добавьте себя в овнеры:

```bash
.addowner <ваш_id>
```

### Проблемы с туннелем

#### `ssh не найден`

```bash
# Termux
pkg install openssh

# Ubuntu/Debian
sudo apt install openssh-client
```

#### Туннель не создаётся

Проверьте переменные окружения:

```bash
export FORELKA_LHR_USER=nokey
export FORELKA_TUNNEL_VERBOSE=1
```

### Общие проблемы

| Проблема | Решение |
|----------|---------|
| Бот не отвечает на команды | Проверьте префикс, права овнера |
| Модули не загружаются | Проверьте синтаксис, зависимости |
| Ошибки в логах | Используйте `.log` для диагностики |
| Проблемы с авторизацией | Удалите сессию, авторизуйтесь заново |
